Security

Security first. Always.

Your financial data deserves the highest protection.

🔐

Row-Level Security

Every database row enforces that only you can access your data — at the database level.

🔑

OAuth 2.0

Sign in with Google or GitHub. No passwords stored. Industry-standard authentication.

🛡️

Encrypted at Rest

⚡

Edge Functions

API keys never reach your browser. All external calls go through secure server-side functions.

Learn more about our security → Click to expand

Row-Level Security (RLS): Every single table in our database has RLS policies enforced at the PostgreSQL level. This means even if a bug existed in our application code, the database itself would reject unauthorized data access.


Authentication: We use Supabase Auth which implements OAuth 2.0 flows. Your Google or GitHub credentials never touch our servers — authentication tokens are issued directly by the provider.


API Key Protection: All stock price and external API calls are made from Supabase Edge Functions running on Deno Deploy. Your browser never sees any API keys.


Cookie Policy: We use only essential session cookies required to keep you logged in. No tracking, no advertising cookies.
